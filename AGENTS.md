# AGENTS.md

本文件是 AI 协作的项目级硬约束入口。所有 easysdd 子工作流默认遵守本文件的所有规则。

## 代码规范

- 安全性 = 正确性 > 最小变更 > 可读性 > 一致性
- 优先做最小必要修改，不擅自扩展需求
- 代码注释和文档默认使用中文
- 复杂实现需要必要注释，避免无意义注释
- 优先复用现有依赖和标准库，禁止擅自引入新第三方依赖
- 涉及架构与分层时，保持当前目录结构与职责边界清晰

## 禁止事项

- 未经确认不要删除文件、推远程、改环境、改 CI、改数据库
- 不要为了追求覆盖率写无业务价值的测试
- 不要绕过真实问题做表面修补
- 不要静默吞没不可恢复错误
- 不要把直接截取的原始知识垃圾式塞进知识库
- `data/presets/` 和 `data/worlds/` 只保留可实际使用的最终成品；不要在这些目录下保留废案、草稿、临时版本、失败实验或未确认可用的中间文件。需要留存过程产物时，放到本次任务目录、`tmp/` 或明确命名的备份目录，并在完成后说明。

## 已知坑

- 角色导入、人物档案、仪表盘显示、全链路测试是当前高关注主线，修改时不要偏离主需求
- 与 LLM / 知识库相关的实现必须基于处理后的知识，而不是直接截取原文堆入
- UI / UX 改动需要关注导入入口、粘贴文字入口、转换进度、仪表盘可视化表达是否直观
- 当前 bash 环境偶发 `/c/Users/Kael: Permission denied` 噪声，判断执行结果时要区分真实失败和环境收尾噪声

## UI 验证要求

- 新增或变更任何可配置项时，必须同步补齐前端配置入口：布尔值提供开关，文本/数字提供输入框，枚举提供下拉或单选；同时接入加载、保存、默认值、禁用态和必要校验，禁止只在后端或示例配置里加字段。
- 配置项若会影响运行时行为，前端文案必须写清楚影响范围和风险；调试/危险/生产敏感开关必须默认采用更安全的值，并在说明中标出。

- 涉及前端视觉/交互/布局/状态同步的改动，必须在真实浏览器里打开对应页面验证，禁止只跑语法检查、接口检查或凭代码推断。
- 前端验证必须截图留证：保存到项目内可追溯路径（例如 `goal-*/screenshots/` 或 `tmp/`），最终汇报必须写明截图路径。
- 截图前必须触发与需求匹配的真实状态：例如 MCP 同步、长上下文追加、导入进度、错误态、空态、加载态、刷新后回填等；不能只截默认空页面。
- 验证时必须同时检查 DOM/可见文本/关键数量：例如卡片数量、当前 active panel、输入输出是否完整可见、是否按顺序追加、是否有遮挡/溢出/重叠。
- 对靶场、仪表盘、任务中心、配置页这类复杂页面，至少验证一个桌面视口；涉及移动或窄屏布局时额外验证窄屏截图。
- 修改“观察能力”相关 UI（靶场画布、MCP 同步、运行 trace、输入输出卡片）时，必须用实际数据跑一遍，确认前端能展示上下文链路，而不是把多轮内容塞进单张长卡。
- 如果浏览器截图工具不可用，必须说明原因，并用可替代证据补足：页面 HTML/DOM 快照、接口返回、日志和最小复现步骤；但后续仍应补截图验证。
- 导入、粘贴文字、进度展示、仪表盘图表类改动需要重点看可理解性和完成闭环。
