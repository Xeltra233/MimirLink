> 状态：初始迁移骨架

## 1. 项目简介

MimirLink 是一个面向 QQ 场景的 Tavern 运行时，重点放在长期记忆、角色生命周期管理和后台可观测性上。

当前仓库已存在若干架构规划与实现计划文档；本文件作为 easysdd 体系下的架构总入口，先建立统一索引，后续再逐步把已确认内容归并为稳定架构描述。

## 2. 核心概念 / 术语表

- **Tavern 运行时**：围绕角色卡、世界书、提示词拼装、消息生成与会话状态组织起来的整体运行逻辑。
- **长期记忆**：以 SQLite 持久化的会话、摘要和角色相关记忆数据，而非一次性内存上下文。
- **角色生命周期管理**：角色导入、角色卡解析、数据库绑定/解绑、迁移与删除等围绕角色状态的管理流程。
- **后台可观测性**：通过 Web 控制面板观察当前角色、世界书、数据库、路由和相关运行状态。
- **后端优先聊天运行时**：把真正影响聊天语义的提示词编排与运行期组装逻辑沉到后端，前端主要负责配置、管理和检查。

## 4. 关键架构决定

当前已确认的高层方向：

- 聊天关键语义应逐步收敛到后端，而不是只放在前端。
- 项目核心不是单纯复刻 SillyTavern UI，而是围绕 QQ 场景构建“记忆 + 生命周期 + 运维”的 Tavern 运行时。
- 前端可以承担配置与观测职责，但不应独占生成关键逻辑。
- 角色卡、世界书、预设、记忆数据库策略需要形成可管理、可观测的统一运行时模型。
